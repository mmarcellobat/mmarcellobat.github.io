JS: Arrays (aka vetores, listas)

Organizar e armazenar dados é um conceito fundamental de programação. Uma maneira de organizar dados na vida real é fazendo listas. Uma lista em JS:

Arrays são a forma de criar listas de JS. Arrays podem armazenar qualquer tipo de dados (incluindo strings, números e booleanos). Como as listas, os vetores são ordenados, o que significa que cada item tem uma posição numérica.

Uma maneira de se criar um vetor é agrupar itens entre colchetes []. Os vetores podem armazenar qualquer tipo de dados, ou seja, podemos ter um vetor que contém todos os mesmos tipos de dados ou um vetor que armazena diferentes tipos de dados, até mesmo outros vetores (aninhados).

Acessando elementos

Cada elemento em um array tem uma posição numerada conhecida como seu índice. Podemos acessar itens individuais usando o índice deles, que é semelhante a fazer referência a um item em uma lista com base na posição do item.

Os arrays em JS são indexados em zero, o que significa que as posições começam a contar a partir de 0 em vez de 1. Portanto, o primeiro item em uma matriz estará na posição 0.

  • cidades[0] acessará o elemento no índice 0 no array cidades. Você pode pensar em cidades[0] como acessando o espaço na memória que contém a string 'Nova Iorque'.

ATENÇÃO! O que acontece se você tentar acessar um índice que está além do último elemento? Sua tentativa resultará em um undefined

Atualizando elementos

Depois de ter acesso a um elemento em um array, você pode atualizar seu valor.

Variáveis declaradas com a palavra-chave const não podem ser reatribuídas. No entanto, elementos em um array declarado com const permanecem modificáveis. Isso significa que podemos alterar o conteúdo de um array de const, mas não podemos reatribuir um novo array ou um valor (tipo) diferente.

Tamanho

Uma das propriedades internas de um array é o seu tamanho (.length) que informa o número de itens no array. Nós acessamos a propriedade .length como fazemos com strings.

Adicionando e removendo elementos

O método .push() nos permite adicionar itens ao final de um array, sem retornos.

O método .pop() nos permite remover o último item de um array e retorna o mesmo.

Outros métodos de arrays

Você pode acessar a documentação de outros métodos de array aqui: Mozilla Developer Network (MDN) array documentation

.pop() e .push() mutam o array no qual eles são chamados. No entanto, há momentos em que não queremos alterar o array original e podemos usar métodos de matriz não-mutantes. Certifique-se de verificar o MDN para entender o comportamento do método que você está usando.

Alguns métodos de matrizes disponíveis para desenvolvedores de JavaScript incluem: .join(), .slice(), .splice(), .shift(), .unshift() e .concat(), entre muitos outros. Usando esses métodos internos, é mais fácil executar algumas tarefas comuns ao trabalhar com arrays.

Por exemplo, .shift() e .unshift() remove ou adiciona, respectivamente, o primeiro elemento da array.

.indexOf() recupera o índice de um elemento no array.

JS: Laços (loops)

Um loop é uma ferramenta de programação que repete um conjunto de instruções até que uma condição especificada, chamada de condição de parada, seja atingida. Como programador, você descobrirá que confia em loops o tempo todo! Você ouvirá o termo genérico iterativo ao se referir a loops; iterar simplesmente significa “repetir”.

Quando precisamos reutilizar uma tarefa em nosso código, geralmente agrupamos essa ação em uma função. Da mesma forma, quando vemos que um processo tem que repetir várias vezes seguidas, escrevemos um loop. Os loops nos permitem criar um código eficiente que automatiza os processos para criar programas escalonáveis e gerenciáveis.

Conforme ilustrado no diagrama a seguir (Fonte: Codecademy, learn javascript: loops), os loops iteram ou repetem uma ação até que uma condição específica seja atendida. Quando a condição é atendida, o loop é interrompido e o computador passa para a próxima parte do programa.

O laço for

Em vez de escrever o mesmo código várias vezes, os loops nos permitem dizer aos computadores para repetir um determinado bloco de código por conta própria. Uma maneira de fornecer essas instruções é com um loop for.

Um loop for contém três expressões separadas por ; dentro dos parênteses:

  • uma inicialização inicia o loop e também pode ser usada para declarar a variável iteradora.
  • uma condição de parada é a condição na qual a variável iteradora é avaliada - se a condição for avaliada como verdadeira, o bloco de códigos será executado e, se for avaliado como falso, o código será interrompido.
  • uma instrução de iteração é usada para atualizar a variável do iterador em cada loop.

O típico loop for inclui uma variável iteradora que geralmente aparece em todas as três expressões. A variável do iterador é inicializada, verificada em relação à condição de parada e atribuída um novo valor em cada iteração de loop. As variáveis de iterador podem ter qualquer nome, mas é recomendável usar um nome de variável descritivo.

Vamos detalhar o exemplo:

  • A condição de parada é conta < 4, o que significa que o loop será executado enquanto a variável do iterador, conta, for menor que 4.
  • A instrução de iteração é conta++. Isso significa que após cada loop, o valor de conta aumentará em 1. Para o primeira iteração, conta será igual a 0, para o segunda iteração será igual a 1 e assim por diante.

O laço reverso com for

Com modificações simples nas expressões, podemos fazer nosso loop rodar para trás! Para executar um loop reverso com for, devemos:

  • Definiir a variável iteradora para o valor desejado mais alto na expressão de inicialização.
  • Definir a condição de parada para quando a variável do iterador for menor que a quantidade desejada.
  • O iterador deve diminuir em intervalos após cada iteração.

Laços e arrays

Laços são muito úteis para iterar sobre estruturas de dados. Por exemplo, podemos usar um loop for para executar a mesma operação em cada elemento em uma matriz. As matrizes armazenam listas de dados, como nomes de clientes ou informações sobre produtos. Imagine que possuímos uma loja e queríamos aumentar o preço de cada produto em nosso catálogo. Isso poderia ser um monte de repetição de código, mas usando um loop for para percorrer a matriz, poderíamos realizar essa tarefa facilmente.

No laço acima, nomeamos nossa variável iteradora i. Essa é uma convenção de nomenclatura de variáveis que você verá em muitos laços. Quando usamos i para iterar através de matrizes, podemos pensar nele como sendo de curta duração para o índice de palavras. Observe como nossa condição de parada verifica se i é menor que animals.length. Lembre-se que os arrays são indexadas com início em zero, o índice do último elemento de uma matriz é equivalente ao .length desse array menos 1. Se tentássemos acessar um elemento no índice de animals.length, teríamos ido longe demais (resultado: undefined)! Com laços for, é mais fácil trabalharmos com elementos em arrays.
